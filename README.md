# 中職即時觀戰小工具 (CPBL Live Tracker)

中職即時觀戰小工具是一個基於 **Chrome Extension Manifest V3** 規範開發的瀏覽器擴充功能。專為中華職棒球迷設計，提供質感暗色調與玻璃擬物化（Glassmorphism）界面的即時球賽數據追蹤面板。

---

## 核心功能介紹 (Features)

1. **即時比分與基本看板 (Live Scoreboard)**
   * 即時同步官方數據，顯示主客隊比分（R）、安打數（H）與失誤數（E）。
   * 即時呈現當前局數與攻守狀態（局上/局下/局中）。
   * 顯示該場比賽的即時天氣、MVP 與勝投/敗投/救援成功之投手資訊。

2. **獨家動態壘包模擬器 (Base Simulation)**
   * **解決 API 滯後問題**：由於官方 API 的壘包狀態存在一個打席的延遲，小工具內置 `simulateCPBLBasesAfterPlay` 演算法，即時剖析文字賽況（如一壘安打、四壞保送等關鍵字），完成投球即點亮壘包。

3. **智慧在壘跑者明細與上壘方法追蹤 (Runner Details)**
   * 顯示一壘、二壘、三壘上的跑手姓名。
   * 自動追溯跑手是透過何種方式上壘（例如：`一壘安打`、`四壞保送`、`趁傳上壘`）。
   * 支援**盜壘與推進追蹤**：當跑者發動盜壘或藉由趁傳推進時，狀態會自動累加顯示（例如：`一壘安打 + 盜壘成功`）。若跑者回本壘得分或出局，會即時從壘包與面板清除。

4. **Due Up 下三棒智慧對齊 (Next Batters Prediction)**
   * 當半局結束（局中狀態）時，自動依據上局最後一個出局者的棒次，精準推算下半局進攻方即將上場的下三棒打者姓名，讓您提前掌握進攻陣容。

5. **即時投打對決數據 (Pitcher & Batter Matchup)**
   * **投手數據**：當前投手的防禦率（ERA）、當局投球數（與累計用球數）。當更換投手時，局球數自動歸零。
   * **打者數據**：當前打者的今日成績（打數-安打數、今日 AVG）。

6. **全場文字播報日誌 (Play-by-Play Logs)**
   * 捲動式清單，完整保存該場比賽自第一球起的所有打席文字敘述，方便隨時回溯賽況。

7. **開賽前一小時桌面提醒 (Desktop Notifications)**
   * 背景常駐 Service Worker，每日定時檢查賽程。當所關注的賽事於一小時後開賽時，將自動推送系統桌面通知提醒您準備觀戰。

8. **富邦悍將首選過濾機制**
   * 自動偵測當日賽事，若包含「富邦悍將」且賽事「未延賽」，小工具開啟時將自動設為預設預覽場次，滿足悍將球迷的首選需求。

## GitHub 下載、打包與安裝教學 (GitHub Download, Packaging & Installation)

由於此小工具尚未上架至 Chrome 線上應用程式商店，若您是從 GitHub 取得此專案，請按照以下步驟進行下載、打包或安裝使用：

### 第一步：取得專案檔案
您可以選擇以下兩種方式之一取得專案原始碼：

* **方法 A：下載 ZIP 壓縮檔（推薦一般使用者）**
  1. 在本 GitHub 專案頁面右上角，點擊綠色的 **「Code」** 按鈕。
  2. 選擇 **「Download ZIP」**。
  3. 下載完成後，將 ZIP 檔案解壓縮至您的電腦中（例如解壓至桌面或我的文件，請記住解壓後的資料夾路徑）。

* **方法 B：使用 Git Clone 指令（推薦開發者）**
  開啟終端機或命令提示字元 (cmd)，執行以下指令將專案複製到本地：
  ```bash
  git clone https://github.com/您的帳號/專案名稱.git
  ```

---

### 第二步：打包與發布（若您需要分享給他人）
若您需要將此擴充功能打包分享給其他球迷，可以使用以下兩種方式：

* **方法 A：簡單打包為 ZIP 壓縮檔（推薦，方便跨平台載入）**
  直接將專案的根目錄資料夾（包含 `manifest.json`、`popup_v3.html` 等檔案的資料夾）進行壓縮，打包成一個 `cpbl_extension.zip`。其他人收到後只需將其解壓縮，並按照下方「安裝步驟」載入該資料夾即可使用。

* **方法 B：在 Chrome 中打包為專屬安裝檔 (.crx)**
  1. 開啟 Chrome 瀏覽器，進入 `chrome://extensions/` 管理頁面。
  2. 啟用右上角的 **「開發者模式」**。
  3. 點擊左上角的 **「打包擴充功能 (Pack extension)」** 按鈕。
  4. 在「擴充功能根目錄」欄位中，選取您的本地資料夾路徑（如 `[您的安裝路徑]`）。
  5. 點擊「打包擴充功能」後，Chrome 會在該資料夾的上一層目錄生成：
     * **`.crx` 檔案**：此為擴充功能安裝包。
     * **`.pem` 檔案**：此為私密金鑰檔案（請妥善保存，後續更新版本時需載入此金鑰以維持相同的 Extension ID）。

---

### 第三步：安裝與使用步驟
1. **開啟擴充功能管理頁面**
   * 打開 Google Chrome 瀏覽器。
   * 在網址列輸入 `chrome://extensions/` 並按下 Enter 鍵，或點擊 Chrome 右上角選單的 `擴充功能 -> 管理擴充功能`。

2. **啟用「開發者模式」**
   * 在擴充功能頁面的右上角，將 **「開發者模式 (Developer mode)」** 的開關切換為**開啟 (On)** 狀態。

3. **載入擴充功能資料夾**
   * 點擊頁面左上角的 **「載入未封裝項目 (Load unpacked)」** 按鈕。
   * 在彈出的資料夾選擇視窗中，選取剛解壓縮出來的專案根目錄（即包含 `manifest.json` 檔案的資料夾）。
   * 點擊「選擇資料夾」按鈕。

4. **完成安裝與釘選**
   * 安裝成功後，Chrome 擴充功能面板會出現 **「CPBL Live Tracker」**。
   * 點擊 Chrome 右上角的「拼圖圖示（擴充功能）」，找到 CPBL Live Tracker 並點擊右側的「圖釘圖示」，將其釘選到瀏覽器工具列，即可一鍵開啟即時觀戰面板！

## 專案核心檔案結構 (Project Structure)

* `manifest.json`：擴充功能的配置描述檔（宣告 MV3 規範、Alarms/Notifications/Storage 權限及背景腳本入口）。
* `popup_v3.html`：小工具主看板面板的 HTML 結構。
* `popup_v3.css`：實現玻璃擬物化視覺、SBO 燈號與鑽石壘包動態定位的樣式表。
* `popup_v3.js`：核心解析與渲染引擎，負責 API 抓取、壘包模擬、跑者追蹤與 DOM 更新。
* `background.js`：背景 Service Worker，用於定時抓取賽程與開賽提醒桌面通知發送。
* `icons/`：存放不同尺寸的中職風格擴充功能圖標。

## 常見問題 (FAQ)

* **Q: 為什麼壘包顏色沒有點亮？**
  * **A**: 請先確認比賽是否已開賽。若賽事進行中但壘包未亮，可能是當打席剛結束，請靜待 API 下一次輪詢同步（預設每 10 秒自動更新一次）。
* **Q: 為什麼開賽提醒通知沒有彈出？**
  * **A**: 請確認您的 Windows 系統通知設定已允許 Google Chrome 傳送通知，且您的擴充功能已常駐背景（可於 `chrome://extensions/` 檢查 background.js 的運行狀態）。
